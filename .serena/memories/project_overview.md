# Project overview
- Purpose: PyTorch implementation of Social LSTM variants (social, obstacle occupancy, vanilla) for pedestrian trajectory forecasting; forked/based on vvanirudh/social-lstm-pytorch. Datasets in `data/train|validation|test` use TrajNet-style txt files.
- Tech stack: Python, PyTorch, NumPy, pandas (data IO), matplotlib (plots/animations), argparse/itertools/random/math/os/pickle/subprocess. No requirements file; install torch/numpy/pandas/matplotlib manually.
- Structure: root scripts for training (`train.py`, `olstm_train.py`, `vlstm_train.py`), inference (`test.py`), validation (`validation.py`), visualization (`visualize.py`), hyperparameter sampling (`hyperparameter.py`), data augmentation (`generator.py`), data utilities (`utils.py` with `DataLoader`), helpers (`helper.py`), grid ops (`grid.py`), model defs (`model.py`, `olstm_model.py`, `vlstm_model.py`), directory bootstrap (`make_directories.sh`), extra tools (`hyperparameter.py`, `visualize.py`). Data folders contain txt trajectories.
- Data handling: `DataLoader` reads txts, pre-processes to `.cpkl` cache, supports train/val/test/infer modes, uses base paths `data/train`, `data/validation`, `data/test`; can sample validation subsets and generate occupancy/social grids.
- Models: Social/Obstacle/Vanilla LSTM classes embed positions, optionally embed social/occupancy tensors, use LSTMCell/GRUCell and linear output heads.
- Outputs: checkpoints under `model/<METHOD>/<LSTM|GRU>`, logs under `log/...`, plots/animations under `plot/...`, results under `result/...`; all created via `make_directories.sh` (also invoked from scripts).
